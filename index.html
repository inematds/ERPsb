Pré-requisitos

O que você precisa antes de começar

Stack Next.js + Prisma + PostgreSQL. Recomendado usar pnpm.

Node.js + pnpm

Node 18+ e pnpm para gerenciar dependências do projeto.

# verificar versões
node -v
pnpm -v

PostgreSQL

Banco de dados via Prisma (local ou serviço gerenciado).

# copiar variáveis de ambiente
cp .env.example .env

Mercado Pago

Credenciais para PIX e conciliação automática de pagamentos.

# preencha no .env
MP_ACCESS_TOKEN="seu-token"
Guia de uso · passo a passo

Rodando o ERPsb localmente

Comandos reais do package.json. O postinstall já gera o Prisma Client automaticamente.

1

Clonar e instalar

Baixe o repositório e instale dependências. O Prisma Client é gerado no postinstall.

git clone https://github.com/inematds/ERPsb.git
cd ERPsb && pnpm install  # roda prisma generate
2

Configurar variáveis

Copie o exemplo e preencha banco, Mercado Pago e Sentry.

cp .env.example .env  # edite DATABASE_URL e demais chaves
3

Subir o banco

Aplique o schema Prisma no PostgreSQL configurado.

pnpm exec prisma migrate dev  # cria as tabelas
4

Rodar em desenvolvimento

Servidor Next.js com Turbopack. Acesse no navegador ou no celular.

pnpm dev  # http://localhost:3000
5

Qualidade e build

Lint, checagem de tipos, testes (Vitest) e build de produção.

pnpm lint && pnpm type-check
pnpm test:run  # testes Vitest
pnpm build     # build de produção
